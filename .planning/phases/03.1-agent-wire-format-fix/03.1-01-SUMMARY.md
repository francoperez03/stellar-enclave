# Phase 03.1 Plan 01: Agent Wire Format Fix Summary

Closed Phase 3 deviation 6 by populating all 8 missing `ShieldedProofWireFormat` public-input fields in the agent's fetch-interceptor using a new dependency-free `decomposePublicInputs()` helper.

## What Was Built

**New module: `packages/agent/src/publicInputs.ts`**

Exports `decomposePublicInputs(publicInputBytes: Uint8Array): ShieldedProofPublicInputs` that splits the 352-byte `proveResult.publicInputBytes` (11 × 32-byte LE BN254 field elements) into the 8 named `ShieldedProofWireFormat` fields:
- 7 decimal u256 strings: `root`, `publicAmount`, `inputNullifiers[0/1]`, `outputCommitment0/1`, `aspMembershipRoot`, `aspNonMembershipRoot`
- 1 big-endian 64-char hex: `extDataHash` (LE bytes reversed then hex-encoded, matching `hashExtData().hex` convention)

Field ordering follows `circuits/src/policyTransaction.circom` lines 39-46 (PI[0]=root, PI[1]=publicAmount, PI[2]=extDataHash, PI[3/4]=nullifiers, PI[5/6]=commitments, PI[7/8]=membership root × 2, PI[9/10]=non-membership root × 2) with invariant guards that throw if the circuit-level duplicates diverge.

**Updated: `packages/agent/src/fetch-interceptor.ts`**

Four surgical edits:
1. Import `decomposePublicInputs` + `ShieldedProofPublicInputs` from `publicInputs.js`
2. Added `extractFixturePublicInputs()` helper supporting PRIMARY (704-char hex at `fixtureEntry.publicInputs`) and LEGACY (pre-decomposed strings inside `fixtureEntry.proof`) shapes
3. Live path: `decomposePublicInputs(proveResult.publicInputBytes)` on `proofPayload`
4. `proofWire` now assigns all 11 `ShieldedProofWireFormat` fields — deviation-6 gap comment deleted

**Updated: `packages/agent/src/__tests__/fetch-interceptor.test.ts`**

- `MOCK_PUBLIC_INPUTS`: deterministic 352-byte fixture with unique per-chunk values, circuit-invariant-compliant (chunks 7/8 equal, 9/10 equal)
- `EXPECTED_DECOMPOSED`: ground-truth decomposed values for assertion
- Extended existing "POSTs /settle with paymentPayload wrapper" test to assert all 8 new fields
- New Phase 03.1 test: PRIMARY-shape `publicInputs` hex flows through `extractFixturePublicInputs` into the wire
